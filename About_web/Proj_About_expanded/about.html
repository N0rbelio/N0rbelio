    Projetos Curriculares

    1. O primeiro grande projeto foi fazer uma aplicação em Python 3 que fizesse uma ligação à API do site ipma.pt para obter a previsão do tempo para uma determinada região escolhida pelo utilizador. O projeto tinha como requisitos fazer o aplicativo na linguagem Python 3. Desenvolver também um UI (User Interface) também em Python 3 que permitisse ao utilizador fazer login e, de seguida, escolher a região, bem como quantos dias de previsão queria (limite de 5 dias, devido à API), armazenar novas contas criadas numa DB (sqlite foi a escolhida) e, por último, dar a possibilidade de utilizar um leitor RFID para o utilizador entrar diretamente na sua conta.

      Usei o Gitlab para guardar a informação do desenvolvimento.



    2. O segundo projeto foi fazer uma aplicação que lesse os conteúdos de uma determinada pasta. O projeto tinha como requisitos fazer o aplicativo usando a linguagem Shell Script. A função era a leitura das pastas e fazer um backup incremental, bem como informar quais foram os últimos ficheiros alterados e por qual utilizador.


    3. Este foi o último e o maior projeto do curso, realizado em grupos de 2, e tinha como objetivo colocarmos todo o conhecimento que adquirimos ao longo do curso. De uma forma muito resumida, tínhamos de criar a rede de uma empresa do zero. Isto envolvia planeamento e configuração de switches, routers e firewall, configuração dos dois servidores com vários serviços, nomeadamente LDAP (suporte de autenticação de Computadores Windows e Linux na rede), Snort (análise de pacotes de rede), VOIP (Voz em IP utilizamos o Asterisk), Mail (RoundCube), entre muitos serviços. Posso dizer que não me importava nada de voltar a fazê-lo.

      Nota: Foram queimadas muitas horas à volta de erros parvos, mas conseguimos fazer o projeto todo apenas usando servidores a correr Linux (únicos da turma).

    Projetos Pessoais

    • A nível pessoal, tenho alguns projetos a decorrer. O primeiro e mais importante é o meu HomeServer/Lab, resumidamente, é um projeto sempre a sofrer alterações. Alguns dos serviços a correr são Servidor de VPN (Wireguard), isto permite-me, em qualquer lugar, ligar-me à minha rede de casa e aceder a algum ficheiro que precise, bem como a vantagem de me proteger se tiver de me ligar a alguma rede pública, WebServer (WordPress, tinha ideia de fazer uma página minha com algumas informações), NextCloud (serve para fazer sync de fotos, vídeos e ficheiros sem estar dependente da google ou outro serviço que eu não tenha o controlo) e, por último, Docker com WebUI do Portainer que permite-me gerir todos os meus Containers ("Mini VirutalMachines"). Todos os serviços acima estão cada um num container para estarem isolados do sistema principal, bem como dos outros serviços.


    • Nota: Já refiz o HomeServer umas 3 vezes tendo como razão trocar inicialmente de FreeBSD para Debian12 e para o atual Kubuntu 23.10 (No Desktop Env).
